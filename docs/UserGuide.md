## 产品简介

### 什么是 OpenTranslator？

OpenTranslator 是一款**完全离线**的桌面翻译工具，基于腾讯混元（Hunyuan）Hy-MT2 大语言翻译模型。所有翻译计算在您的本地电脑上完成，无需联网，保护您的隐私和数据安全。

### 核心特性

| 特性 | 说明 |
|------|------|
| ✅ 完全离线 | 无需联网，所有翻译在本地完成 |
| ✅ 多语言支持 | 支持 37 种语言互译 |
| ✅ 划词翻译 | 选中文字一键翻译，悬浮窗显示结果 |
| ✅ 原地替换 | 翻译后直接替换原文，高效编辑 |
| ✅ 翻译历史 | 自动保存翻译记录，支持搜索和收藏 |
| ✅ 智能检测 | 自动识别源语言 |
| ✅ 硬件加速 | 支持 CPU 和 GPU（CUDA）加速 |
| ✅ 跨应用使用 | 全局热键，可在任何应用中使用 |

### 支持的语言

OpenTranslator 支持 **37 种语言**，包括：

**常用语言**：
- 中文（简体/繁体）、英语、日语、韩语
- 法语、德语、西班牙语、葡萄牙语、俄语

**亚洲语言**：
- 泰语、越南语、印尼语、马来语、菲律宾语
- 印地语、孟加拉语、泰米尔语、泰卢固语
- 藏语、维吾尔语、蒙古语、哈萨克语、粤语

**其他语言**：
- 阿拉伯语、希伯来语、波斯语、土耳其语
- 意大利语、荷兰语、波兰语、捷克语、乌克兰语
- 高棉语、缅甸语、乌尔都语、古吉拉特语、马拉地语

## 系统要求

### 最低配置

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 或 Windows 11（64位） |
| 处理器 | 任意 x64 处理器 |
| 内存 | **8 GB**（使用 1.8B 模型） |
| 磁盘空间 | 2 GB（程序 + 模型文件） |
| 显卡 | 可选（NVIDIA 显卡可加速） |

### 推荐配置

| 项目 | 要求 | 说明 |
|------|------|------|
| 内存 | **16 GB** | 可使用更大的 7B 模型，翻译质量更高 |
| 显卡 | NVIDIA GPU，4GB+ 显存 | CUDA 加速，翻译速度提升 3-5 倍 |

### 显存与模型对应表

| 显存大小 | 推荐模型 | 翻译速度 |
|----------|----------|----------|
| 无显卡 / 2GB以下 | Hy-MT2-1.8B-1.25bit | 较慢（CPU推理） |
| 2GB | Hy-MT2-1.8B-2bit | 中等 |
| 4GB+ | Hy-MT2-1.8B-Q4_K_M | 快速 |
| 8GB+ | Hy-MT2-7B-Q4_K_M | 快速 + 高质量 |
| 16GB+ | Hy-MT2-30B-A3B | 最高质量 |

## 快速入门

### 启动程序

双击 `OpenTranslator.exe` 启动程序。

首次启动时，程序会：
1. 自动检测电脑硬件配置
2. 扫描 models 目录查找模型文件
3. 加载找到的第一个模型（或配置中的默认模型）
4. 初始化翻译引擎

**首次加载模型可能需要 10-30 秒**，请耐心等待。

### 主窗口界面

启动后显示主窗口，布局如下：

```
┌────────────────────────────────────────────────────┐
│  🌐 OpenTranslator                         ⚙ 设置  │
├────────────────────────────────────────────────────┤
│  [源语言 ▼]         ⇄          [目标语言 ▼]        │
├─────────────────────┬──────────────────────────────┤
│                     │                              │
│   输入原文          │        显示译文              │

└────────────────────────────────────────────────────┘
```

### 第一次翻译

**步骤演示**：

1. **输入文本**：在左侧文本框输入或粘贴要翻译的文字
   ```
   Hello, how are you today?
   ```

2. **选择语言**：
   - 源语言：选择 **"自动检测"**（程序会自动识别英语）
   - 目标语言：选择 **"中文"**

3. **点击翻译**：点击右下角的 **"翻译"** 按钮

4. **查看结果**：右侧显示译文
   ```
   你好，今天过得怎么样？
   ```

5. **复制译文**：点击右侧的 **📋** 按钮，复制译文到剪贴板

## 功能详解

### 1. 主窗口翻译

这是最基本的翻译方式，适合批量翻译或长文本翻译。

**操作步骤**：

1. 输入或粘贴原文到左侧文本框
2. 选择源语言和目标语言
3. 点击 **"翻译"** 按钮
4. 译文显示在右侧
5. 可点击复制按钮获取译文

**提示**：
- 源语言选择 **"自动检测"**，程序会自动识别语言类型
- 支持翻译长文本，上下文窗口最大 4096 tokens（约 3000-4000 字）
- 翻译耗时显示在底部，可评估速度

### 2. 划词翻译（快捷高效）

**划词翻译** 是最常用的功能，可在任何应用中快速翻译选中的文字。

**使用场景**：
- 浏览网页时翻译外文内容
- 阅读文档时翻译段落
- 聊天软件中翻译消息

**操作步骤**：

1. **选中文字**：在任何应用中用鼠标选中要翻译的文字
   ```
   例如在浏览器中选中: "Good morning!"
   ```

2. **按快捷键**：按下 **`Ctrl + Alt + B`**
   - 程序自动获取选中的文字
   - 开始翻译

3. **查看结果**：悬浮窗自动显示翻译结果
   ```
   ┌──────────────────────────────┐
   │  划词翻译                ✕  │
   ├──────────────────────────────┤
   │  [自动检测 ▼]  →  [中文 ▼]  │
   ├──────────────────────────────┤
   │  原文                        │
   │  Good morning!               │
   ├──────────────────────────────┤
   │  译文                        │
   │  早上好！                    │
   ├──────────────────────────────┤
   │  推理耗时: 120ms   [复制]    │
   └──────────────────────────────┘
   ```

4. **复制译文**：点击悬浮窗中的 **"复制"** 按钮

5. **关闭悬浮窗**：点击 **✕** 或按 `Esc` 关闭

**注意事项**：
- 确保程序正在运行（可在后台运行）
- 悬浮窗会自动出现在屏幕合适位置
- 支持在浏览器、Word、PDF阅读器、聊天软件等应用中使用

### 3. 原地替换翻译

**原地替换** 功能会将选中的文字替换为翻译结果，适合编辑文档场景。

**使用场景**：
- 编辑外文文档时，直接替换为中文
- 编写邮件时翻译替换
- 编写代码注释时翻译

**操作步骤**：

1. **选中文字**：在编辑器中选中要翻译替换的文字
   ```
   例如在 Word 中选中: "Thank you for your help."
   ```

2. **按快捷键**：按下 **`Alt + V`**
   - 程序翻译选中的文字
   - 自动将译文粘贴替换原文

3. **查看结果**：选中的文字已被替换为译文
   ```
   感谢您的帮助。
   ```

**注意事项**：
- 需要目标应用支持粘贴操作
- 原文会被直接替换，无法恢复（建议重要文档先备份）
- 剪贴板原有内容会被覆盖

### 4. 翻译历史

程序自动保存每次翻译记录，方便回顾和查找。

**查看历史**：

主窗口下方显示历史记录列表：

```
┌────────────────────────────────────────────────────┐
│  翻译历史                        🔍 [搜索框]       │
├────────────────────────────────────────────────────┤
│  ⭐  14:30  Hello → 你好                           │
│       14:25  Good morning → 早上好                 │
│       14:20  Thank you → 谢谢                      │
├────────────────────────────────────────────────────┤
│  [加载更多]                        [清空历史]      │
└────────────────────────────────────────────────────┘
```

**历史功能**：

| 操作 | 说明 |
|------|------|
| 点击记录 | 自动加载到编辑区，可重新翻译 |
| ⭐ 收藏 | 标记重要翻译，不会被清空 |
| 🔍 搜索 | 输入关键词搜索历史记录 |
| 🗑 删除 | 删除单条记录 |
| 清空历史 | 清除所有未收藏的记录 |

### 5. 语言交换

快速交换源语言和目标语言，实现反向翻译。

**操作**：点击语言选择栏中间的 **⇄** 按钮

**效果**：
- 源语言变为目标语言
- 目标语言变为源语言
- 原文和译文也会交换位置

**注意**：如果源语言是 **"自动检测"**，交换功能不可用。

### 6. 模型切换

如果下载了多个模型，可以随时切换使用。

**切换方式**：

主窗口底部选择模型：
```
状态: 就绪    [Hy-MT2-1.8B-Q4_K_M ▼]    [翻译]
```

点击下拉框，选择其他模型：
- 切换到更大模型 → 翻译质量提升，速度稍慢
- 切换到更小模型 → 翻译速度提升，质量稍降

切换模型时，状态栏显示 **"正在切换到 XXX..."**，完成后显示 **"XXX 已就绪"**。

| MainWindowLayout | 布局（0=左右，1=上下） |

## 常见问题

### Q1: 程序启动后显示 "未找到模型"

**原因**：models 目录中没有模型文件。

**解决方法**：
1. 确认已下载模型文件（如 `Hy-MT2-1.8B-Q4_K_M.gguf`）
2. 将模型文件放入 `OpenTranslator/models/` 目录
3. 重启程序

**验证**：
```
正确的位置：
OpenTranslator/models/Hy-MT2-1.8B-Q4_K_M.gguf

错误的位置：
OpenTranslator/Hy-MT2-1.8B-Q4_K_M.gguf  ← 不要放这里
```

### Q2: 程序启动很慢（超过 30 秒）

**原因**：模型加载需要时间，特别是大模型或使用 CPU 模式。

**解决方法**：
- 使用更小的模型（如 2bit 版本）
- 如果有 NVIDIA 显卡，确保显卡驱动已安装
- 关闭其他占用内存的程序

### Q3: 翻译结果不理想

**原因**：模型翻译质量受多种因素影响。

**优化建议**：
- 尝试使用更大的模型（如 7B 版本）
- 确保源语言选择正确（或使用自动检测）
- 对于专业术语较多的文本，翻译质量可能下降
- 长文本翻译时，分段翻译可能效果更好

### Q4: 划词翻译快捷键不起作用

**可能原因**：

1. **程序未运行**：确保程序正在运行（可在后台）

2. **热键冲突**：其他软件占用了相同快捷键
   - 检查状态栏是否有热键注册失败提示
   - 尝试关闭其他可能有热键的软件

3. **选中文本失败**：某些应用不支持复制选中内容
   - 尝试手动复制后，在主窗口粘贴翻译

### Q5: 翻译速度很慢

**原因分析**：

| 情况 | 原因 | 解决方法 |
|------|------|----------|
| CPU 模式慢 | CPU 推理速度有限 | 使用更小模型，或启用 GPU |
| GPU 不生效 | CUDA 未正确配置 | 检查显卡驱动，确保是 NVIDIA GPU |
| 内存不足 | 系统频繁交换内存 | 关闭其他程序，释放内存 |

**优化建议**：

1. **使用 GPU 加速**：
   - 安装 NVIDIA 显卡驱动
   - 显存 ≥ 2GB 可启用 CUDA 加速
   - 速度可提升 3-5 倍

2. **选择合适的模型**：
   - 低配电脑：使用 2bit 或 1.25bit 模型
   - 中配电脑：使用 Q4_K_M 模型（推荐）
   - 高配电脑：使用 7B 模型（高质量）

3. **释放系统资源**：
   - 关闭浏览器、视频等占用内存的程序
   - 确保内存占用不超过 70%

### Q6: 剪贴板操作失败

**原因**：某些应用限制剪贴板访问，或剪贴板被其他程序占用。

**解决方法**：
- 等待几秒后重试
- 关闭可能占用剪贴板的程序（如截图软件）
- 手动复制粘贴到主窗口翻译

### Q7: 如何卸载？

OpenTranslator 是绿色软件，无需安装，卸载方法：

1. 关闭程序
2. 删除整个 `OpenTranslator` 目录
3. （可选）删除用户数据目录（如果有）

卸载后不会留下注册表项或系统残留文件。

### Q8: 可以在 Mac 或 Linux 上使用吗？

**当前版本**：仅支持 Windows 10/11（64位）。

**未来计划**：可能会推出跨平台版本。
